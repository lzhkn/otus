Разница между образом и контейнером

Образ (image) — это шаблон, набор файловой системы и метаданных, из которого создаются контейнеры. Он неизменяем и служит исходной точкой.

Контейнер — это запущенный экземпляр образа, имеющий собственный слой изменений (writable layer), процессы, сетевые настройки и состояние.

Можно ли собрать ядро в контейнере?

Да, собрать ядро можно, но:

контейнер использует ядро хост-системы и не может подменить его сам по себе;

Компилировать ядро можно, устанавливать и использовать — нельзя, контейнер не имеет собственного ядра.
